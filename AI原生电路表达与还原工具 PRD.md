# AI 原生电路表达与还原工具 PRD

文档版本：v2.1  
文档状态：当前产品方向  
整理日期：2026-04-14  
交换规范入口：[exchange.md](F:/WORKSPACE/EASYAnalyse/exchange.md)

## 1. 一句话定义

这不是一个把传统原理图搬进浏览器的工具，而是一个让 AI 与工程师围绕“器件 + 端子标签 + 方向关系”高效沟通硬件结构的工作台。

## 2. 背景与问题

当前 AI 参与硬件交流时主要卡在三件事上：

1. AI 很难稳定生成真实可用的拓扑式电路 JSON，常见结果是器件虚构、连线随意、结构失真。
2. 即便逻辑关系部分正确，导线规划也经常把图面拖入混乱，阅读成本极高。
3. 即便去掉导线，如果还要求 AI 维护独立 signal 对象和端子绑定，仍然会增加生成和修改复杂度。

所以产品不应继续把“画线”或“维护中间信号层”当成中心任务，而应把“表达真实器件关系”当成中心任务。

## 3. 产品目标

### 3.1 核心目标

1. 让 AI 只需输出语义化 JSON 就能表达硬件结构。
2. 让端子的标签本身直接承担连接语义，标签一致即连通。
3. 让应用基于标签关系生成清晰、宽松、可读的器件布局。
4. 让用户通过颜色、标签、上下游高亮快速看懂连接关系。
5. 让用户在不处理导线路由的前提下完成理解、审查和修改。
6. 让项目在规范、运行时、样例、UI 上保持单一方向，不再新旧混杂。

### 3.2 成功标准

1. AI 生成的 JSON 不需要包含导线、节点或独立信号对象也能被应用正确渲染。
2. 同一标签的端子在图上具有一致颜色和明确标识。
3. 点击器件时，相关输入设备和输出设备可以清晰区分。
4. 双击器件时，用户可以通过聚焦动画快速理解它的局部结构。
5. 根目录文档、schema、示例文件、应用行为彼此一致。

## 4. 非目标

当前阶段不追求：

1. 传统 EDA 式导线路由编辑。
2. 电路仿真。
3. PCB 生成。
4. 完整元件库和封装库。
5. 用几何连线或独立信号树来复刻传统原理图细节。

## 5. 目标用户

1. 需要和 AI 协作定义硬件结构的工程师。
2. 需要快速表达方案的学生、创客和研发人员。
3. 需要审查 AI 生成硬件方案是否靠谱的人。

## 6. 核心产品原则

### 6.1 语义优先

连接关系由端子标签表达，而不是由线段拓扑或 signal 对象表达。

### 6.2 器件优先

画布的主角是器件与模块，不是导线、节点或信号中间层。

### 6.3 可读性优先

布局目标是让相连者相近，同时保持整体宽松、整洁、易读。

### 6.4 交互优先于布线

用户理解关系主要依赖：

- 颜色
- 标签
- 高亮
- 聚焦动画

而不是依赖复杂走线。

## 7. 信息模型

### 7.1 `document`

记录文档标题、描述、来源、时间等元信息。

### 7.2 `devices`

定义图中的器件、模块和基础元件。每个设备内嵌多个 `terminals`。

### 7.3 `terminals`

端子需要能够表达：

- 名称
- 标签
- 方向
- 所在边
- 顺序

其中标签就是连接标签。

### 7.4 `view`

定义画布与布局，不承载连接真相。

## 8. 关键交互

### 8.1 画布显示

1. 画布区域应明显大于其他区域。
2. 画布背景保持纯色网格。
3. 画布是无限平面。
4. 画布中需要有坐标轴。
5. 页面其他背景可以有轻微动态感，表现为流动云层，而不是侵入画布。

### 8.2 标签显示

1. 每个端子以彩色点显示。
2. 相同标签使用相同颜色。
3. 点旁边显示简洁文字标签，如 `SCL`、`GND`、`TX`。
4. 编辑标签时应有快捷索引和过滤建议。

### 8.3 选择器件

1. 点击器件时，当前器件高亮。
2. 与其输入关系相关的器件高亮为红色语义。
3. 与其输出关系相关的器件高亮为绿色语义。

### 8.4 聚焦器件

1. 双击器件时，目标器件平滑移动到视图中心。
2. 与其输入相关的设备飞到左侧。
3. 与其输出相关的设备飞到右侧。
4. 动画需要连续、干净，不应像切页或跳变。

### 8.5 视角控制

1. 用户可以在无限画布中平移和缩放。
2. 用户应能通过快捷键回到坐标原点。

### 8.6 属性编辑

右侧保留简洁属性区，用于编辑：

- 文档信息
- 端子信息
- 器件信息
- 视图设置

端子编辑区应位于属性面板顶部，并支持独立滚动。

## 9. 界面原则

1. 整体界面以白色、浅灰、细边界为主。
2. 画布是主区域，属性面板是辅助区域。
3. 去掉与当前任务无关的文案、面板和旧交互控件。
4. 风格参考现代 EDA 工具，保持克制、简洁、专业。

## 10. 验收标准

### 10.1 规范层

1. `exchange.md` 与正式 schema 描述同一套 `4.0.0` 语义模型。
2. 示例文件只保留现行语义格式。

### 10.2 运行时

1. 编辑器内部不再把拓扑对象作为主模型。
2. 不再依赖导线、节点、折点或独立 signal 对象来表达连接关系。

### 10.3 UI

1. 画布中不显示旧式导线拓扑。
2. 端子颜色、标签、上下游高亮和聚焦动画可正常工作。
3. 页面主次分明，画布面积明显提升。
4. 属性面板独立滚动。
5. 端子设置位于属性顶部。

### 10.4 工程质量

1. 构建通过。
2. Lint 通过。
3. Rust 校验测试通过。
4. 桌面应用打包通过。

## 11. 当前结论

EASYAnalyse 的核心价值不是“替代 EDA 画线”，而是“把硬件结构表达降低到 AI 和工程师都更容易正确处理的层级”。  
因此，产品应长期坚持语义优先、器件优先、可读性优先，并继续压缩一切非必要的中间表达层。
