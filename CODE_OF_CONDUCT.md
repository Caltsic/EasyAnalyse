# Code Of Conduct / 行为准则

## 中文

EASYAnalyse 社区以专业、尊重、可验证的工程协作为基本原则。

### 我们鼓励

- 用事实、复现步骤、代码引用和验证结果讨论问题。
- 对技术方案提出清晰、可执行的改进建议。
- 尊重不同经验背景的贡献者。
- 在讨论硬件安全、数据安全和模型行为时保持谨慎。
- 公开承认不确定性，并把风险写清楚。

### 我们不接受

- 人身攻击、侮辱、骚扰、歧视或威胁。
- 故意制造混乱、刷屏、恶意误导或重复无效争论。
- 未经许可公开他人的隐私、凭据、商业机密或未公开硬件设计。
- 在公开 issue 中披露可利用安全漏洞细节。
- 绕过项目权限、CI、review 或发布流程。

### 处理方式

Maintainer 可以对违反行为准则的内容进行提醒、编辑、隐藏、关闭、限制参与或移除访问权限。安全和隐私相关问题优先按 [SECURITY.md](SECURITY.md) 处理。

## English

The EASYAnalyse community is based on professional, respectful, and verifiable engineering collaboration.

### We Encourage

- Discussing issues with facts, reproduction steps, code references, and validation results.
- Giving clear and actionable technical feedback.
- Respecting contributors with different levels of experience.
- Being cautious when discussing hardware safety, data security, and model behavior.
- Stating uncertainty and documenting risks explicitly.

### We Do Not Accept

- Personal attacks, insults, harassment, discrimination, or threats.
- Deliberate disruption, spam, misleading behavior, or repeated unproductive arguments.
- Publishing another person’s private information, credentials, trade secrets, or unreleased hardware designs without permission.
- Disclosing exploitable security vulnerability details in public issues.
- Bypassing project permissions, CI, review, or release processes.

### Enforcement

Maintainers may warn, edit, hide, close, restrict participation, or remove access for conduct violations. Security and privacy issues are handled first through [SECURITY.md](SECURITY.md).
