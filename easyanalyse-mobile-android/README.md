# EASYAnalyse Mobile Android

Native Android read-only circuit viewer for EASYAnalyse desktop snapshots.

## Architecture

- Kotlin + Jetpack Compose for app chrome, scan screen, toolbar, and details panel.
- Native Android `View` + `Canvas` for the circuit drawing surface.
- No WebView and no browser-rendered circuit canvas.
- Landscape-only first version. The manifest locks `MainActivity` to landscape and the activity also requests landscape at startup.
- The app reads `mobile-render-v1` snapshots generated by the Windows desktop app. AI-generated EASYAnalyse JSON does not change for mobile.
- Device focus uses precomputed snapshot relations to dim unrelated content and fit related devices/networks into the visible landscape area.
- Common device symbols are rendered by `DeviceSymbolRenderer` instead of drawing every component as a generic rectangle.

## Snapshot Flow

1. Windows desktop validates and normalizes the standard EASYAnalyse document.
2. Desktop derives a mobile render snapshot from the normalized document.
3. The LAN share server exposes the snapshot at `/api/mobile/snapshot/{token}`.
4. The Android app scans a QR code or receives an `easyanalyse://open?url=...` deep link.
5. The native canvas renders the snapshot with fit-to-view, pan, pinch zoom, click selection, relation focus, and common device symbols.

## Local Build Notes

This repository includes Gradle project files, but this machine currently does not expose a `gradle` or `gradlew` command. If Android Studio is installed, open this directory directly. Otherwise add a Gradle wrapper or install Gradle before running:

```powershell
Copy-Item local.properties.example local.properties
.\gradlew.bat :app:assembleDebug
```

The app needs LAN HTTP access, so cleartext traffic is enabled intentionally for the local desktop share server.

## Release Signing

This project reads Android release signing data from `easyanalyse-mobile-android/keystore.properties`
or from these environment variables:

- `EA_ANDROID_KEYSTORE_PATH`
- `EA_ANDROID_KEYSTORE_PASSWORD`
- `EA_ANDROID_KEY_ALIAS`
- `EA_ANDROID_KEY_PASSWORD`

To bootstrap a local release keystore on Windows:

```powershell
powershell -ExecutionPolicy Bypass -File ..\scripts\create_android_release_keystore.ps1
```

That script creates:

- `easyanalyse-mobile-android/signing/easyanalyse-release.jks`
- `easyanalyse-mobile-android/keystore.properties`

Both are ignored by Git. After that, a signed release build is produced with:

```powershell
.\gradlew.bat :app:assembleRelease
```
